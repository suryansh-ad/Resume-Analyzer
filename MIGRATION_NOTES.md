# Fresherr Next.js Migration Notes

## Summary

Fresherr has been migrated from a Vite React SPA plus Express API server to a root Next.js App Router application. The production runtime now serves pages and API endpoints from Next.js while preserving the original resume upload, text extraction, Supabase authentication, ATS scoring, AI analysis, and report download behavior.

## Rendering Strategy

- `/`: Static Next page with a client analyzer island. This gives the homepage SEO-ready HTML while preserving Supabase auth, file upload, local analysis history, and dashboard interactivity.
- `/about`: Static page. The content is stable and benefits from prerendering.
- `/auth/reset-password`: Static route with client-side Supabase recovery handling. Supabase recovery tokens are browser-session dependent.
- `/api/*`: Node.js route handlers. Upload parsing, PDF/DOCX extraction, Supabase admin calls, and AI providers require Node runtime.

## API Endpoints Preserved

- `GET /api/health`
- `POST /api/auth/check-email`
- `POST /api/resume/upload`
- `POST /api/resume/analyze`

## SEO Improvements

- App Router metadata for title, description, canonical URL, Open Graph, Twitter cards, robots, and Google verification.
- `robots.txt` generated by `app/robots.js`.
- `sitemap.xml` generated by `app/sitemap.js`.
- JSON-LD for Organization, SoftwareApplication, FAQ, and BreadcrumbList.
- Target keywords included naturally in metadata: ATS Resume Checker, Resume Analyzer, Resume Score Checker, Free ATS Checker, AI Resume Review, Resume Optimization Tool.

## Performance Improvements

- Next font optimization replaces CSS `@import` for Sora.
- Logo images use `next/image`.
- The dashboard is dynamically imported and loaded only when needed.
- Browser-only logic is contained in client components to avoid hydration errors.
- Heavy report/chart code stays out of the initial server-rendered page path.

## Railway Deployment

- Build command: `npm run build`
- Start command: `npm run start`
- Required Node version: `>=20.9.0`
- Required environment variables are documented in `.env.example`.

## Post-Migration Checklist

- Add production `https://fresherr.in` and local/dev tunnel URLs to Supabase Auth redirect URLs.
- Set `NEXT_PUBLIC_SITE_URL=https://fresherr.in` in Railway.
- Configure at least one AI provider key.
- Confirm uploads work with PDF and DOCX files in production.
- Confirm forgot-password and reset-password emails redirect to `/auth/reset-password`.
- Submit `/sitemap.xml` in Google Search Console.
